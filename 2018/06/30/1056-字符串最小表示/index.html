1056 字符串最小表示

发表于 2018-06-30 | 分类于 CodeUp.cn编程题详解

题目描述

把一个长为len的字符串围成一个圈，然后以任意一个字符作为起点，都会产生一个长为len的字符串，字符串的最小表示就是所有字符串中字典序最小的那个。 例如字符串alabala，将它围成一个圈后，根据上面的规则会形成以下新的字符串：

labalaa

abalaal

balaala

alaalab

laalaba

aalabal

在这所有7个字符串中，字典序最小的是aalabal，它的第一个字母在原字符串中的位置是6。（位置从0开始算） 现在给定你一个字符串，请你找出其最小表示的第一个字母在原字符串中的位置。如果字符串最小表示有多个，那么输出第一个字母在原字符串中位置最小的。

输入

输入的第一行是一个整数t，表示有t组测试数据。
接下来t行，每行先输入一个整数l（5<=l<=100000），表示原字符串的长度，然后输入一个字符串，表示原字符串。字符串中只包含小写字母。

输出

对于每组输入，输出原字符串最小表示的第一个字母在原字符串中的位置。

方法二，首先明确，只需要找到最小字符串开始的下标即可，不用改变字符串，对于下标的变化只需要对字符串长度取余即可，保证其在0 ~ len-1范围内循环变化。

首先令i = 0, j = 1。

当s[i] > s[j]时，说明从下标i开始的字符串不可能是最小字符串，所有下标i被淘汰，使i = j++，这里是为了保证i < j；

当s[i] < s[j]时，说明从下标j开始的字符串不可能是最小字符串，所有下标j被淘汰，j++后移一位；

经过上面两步的处理，最终会有s[i] == s[j]，且以i+1 ~ j-1字符开头的字符串都不可能是最小字符串了。

于是讨论s[i] == s[j]时，令k = 1，开始比较s[i+k]与s[j+k]，直到遇见第一个s[i+k] != s[j+k]：

当s[i+k] > s[j+k]时，说明从下标i开始的字符串不可能是最小字符串，i被淘汰。同时i+1 ~ i+k都不可能成为最小字符串的开头，

因为，以这些下标开头而以i+k结尾的部分不可能是最小字符串的前缀，

因为，以从j+k往前相同长度的部分作为前缀的字符串一定比它小。（s[i+k] > s[j+k] 且 之前的字符直到i和j都是相等字符）。

所以，把i更新为i+k+1。

同理，当s[i+k] > s[j+k]时，j更新到j+k+1。

最后，i和j有一个扫描到了字符串结尾，则另一个就是结果。
